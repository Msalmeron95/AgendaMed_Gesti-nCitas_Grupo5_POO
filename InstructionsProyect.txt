Ejercicio a desarrollar
Crear un programa que ayude a gestionar citas médicas, para ello tiene 3 tipos diferentes de usuarios, el administrador, los doctores y los pacientes, cada uno debe tener un email y una contraseña para poder iniciar sesión, el programa deberá solicitar los datos para ingresar al sistema, y dependiendo del usuario que ingrese deberá ver un menú diferente, si inicia sesión el doctor podrá ver las siguientes opciones, ver todas las citas que tiene, ver todos sus pacientes, ver el historial médico de cada paciente, cancelar citas y colocando el motivo por el cual se cancela, el paciente podrá ver los doctores disponible por especialidad y podrá agendar una cita siempre y cuando el doctor tendrá espacio para atenderle ese día, los pacientes podrán ver todas las citas que han tenido y su historial personal. Si el usuario que ingresa es el administrador, podrá gestionar los pagos de los doctores, podrá agregar nuevos doctores, agregar las especialidades de los doctores, ver todas las citas, ver todos los pacientes y los doctores

Como no estamos trabajando con bases de datos, el programa debe simular que ya existen usuarios registrados, para ello debe crear una lista con al menos 1 administrador, 5 doctores y 10 pacientes. Con eso a ese listado se verificarán los datos al intentar iniciar sesión.

Puntos a Evaluar
El programa debe realizar correctamente lo que se indicó en el ejercicio
El programa debe hacer uso de Spectre.Console
El programa debe controlar las excepciones, debe integrar como mínimo una excepción personalizada
El código debe cumplir con buenas prácticas y debe ser ordenado
El programa debe integrar Enums
El programa debe hacer uso de Interfaces
El programa debe hacer uso de Records
El programa debe tener las clases respectivas para controlar todo lo que se le solicita en el enunciado del proyecto
Puede aplicar todo lo que hemos visto en el curso en caso de ser necesario como herencia, polimorfismo, etc.
Se evaluará la puntualidad de la entrega. Si se entrega después del tiempo estipulado recibirá una sanción de puntos en la calificación.
El repositorio que entreguen debe tener constancia que todo el grupo trabajó.
La tarea la tiene que entregar en dos formatos:
Un archivo comprimido con el nombre del grupo (grupo 1, grupo 2, etc.), debe llevar dentro del archivo ZIP un archivo txt con los nombres completos de los integrantes del equipo, el carnet de cada uno y el código completo, además deberá compartir el script de la base de datos para hacer las pruebas.
El segundo formato es el repositorio de GitHub debe compartir el código que trabajaron para poder ver como se trabajó (pueden incluir dentro del ZIP el link del repositorio o lo comparten aparte, lo importante es que vaya el link para ver el código desde ahí).
